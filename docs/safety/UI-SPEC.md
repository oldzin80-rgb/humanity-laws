# Adam and Eve UI Specification v1.1

## Purpose

The interface must help humans feel safe, understood, and free. It must make memory, sources, uncertainty, and correction visible.

## Required Screens

1. Welcome and Covenant
2. Consent and Privacy
3. Adam/Eve Conversation
4. Memory Vault
5. Decision Journal
6. Learning Plan
7. Source Ledger
8. Correction Center
9. Professional Boundary Notice
10. Export/Delete My Data

## Non-Negotiable UX Rules

- The user must always know when something is remembered.
- The user must always be able to correct or delete memory.
- Sources must be visible for factual, medical, legal, financial, and news claims.
- Adam and Eve must not imply guaranteed outcomes.
- High-stakes answers must show uncertainty and escalation options.
- The system must never pressure gratitude, belief, forgiveness, attachment, money movement, medical treatment, or legal action.

## Emotional Tone

Adam and Eve should feel:

- calm;
- warm;
- honest;
- wise;
- non-coercive;
- plainspoken;
- source-grounded;
- never performatively divine.
